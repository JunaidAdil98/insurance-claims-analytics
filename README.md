# Auto Insurance Claims Analytics

**Diagnosing why a property-and-casualty auto book is running unprofitably — where losses concentrate, what drives claim cost and frequency, and how recent accident years are developing.**

> **Data disclosure:** the dataset is **synthetic**, generated by [`analysis/generate_data.py`](analysis/generate_data.py). Every figure in the analysis and dashboard is the *real output* of the pipeline run on that data. The methodology transfers directly to production claims data.

## The business question

A simulated auto book of **9,000 policies (FY2021–FY2024)** is posting a **96.6% loss ratio** — for every premium dollar earned, ~97¢ goes back out in losses before expenses, i.e. the book is unprofitable. The analyst's job: find *where* the losses sit and *what drives them*, so underwriting and pricing can act.

## Headline findings

| Metric | Value |
|---|---|
| Policies / Claims | 9,000 / 2,305 |
| Claim frequency | 25.6% (claims per policy) |
| Average severity | $5,612 per claim |
| Earned premium / Incurred | $13.4M / $12.9M |
| **Portfolio loss ratio** | **96.6%** |

- **Vancouver is the worst region at a 112% loss ratio** — losses exceed premium outright. Toronto (97%) and Montreal (96%) follow. The three urban books are where repricing has the most leverage.
- **Comprehensive coverage runs hottest (102%)**, Liability the most profitable (68%) — a signal that comprehensive is underpriced for the risk it carries.
- **Young drivers (16–24) file the most claims** per policy; frequency falls through middle age and ticks back up for seniors.
- **Liability claims are the most expensive on average (~$8k)**; glass/minor claims the cheapest (~$1.7k).
- **Accident year 2024 is only ~83% developed** — open claims mean ultimate losses will rise, which matters for reserving.

## Models

- **Severity regression** (log-linear, scikit-learn): R² ≈ 0.30. Claim *type* dominates cost — glass/minor −68%, liability +39%, theft +39% versus baseline; urban regions add a further premium. Modest R² is expected: severity is inherently noisy, and the value is in *directional, interpretable* drivers, not point prediction.
- **Frequency model** (logistic): AUC ≈ 0.61. Prior claims and urban region raise the odds of a claim; older drivers and liability-only cover lower them. Frequency is hard to predict at the individual level — an honest AUC here is itself a finding.

## How it's built

```
analysis/
  generate_data.py    # synthetic portfolio with realistic risk structure
  analysis.py         # descriptive + diagnostic + cohort + regression -> docs/dashboard_data.json
  build_dashboard.py  # embeds the JSON into a self-contained dashboard
sql/
  schema.sql          # T-SQL warehouse schema
  analysis_queries.sql# the same diagnostics + cohort + NTILE segmentation in T-SQL
data/                 # generated CSVs (customers, policies, claims)
docs/
  index.html          # the dashboard (this is what GitHub Pages serves)
  dashboard_data.json
```

**Stack:** Python (pandas, scikit-learn) · T-SQL (SQL Server) · Chart.js · GitHub Pages.

## Run it yourself

```bash
pip install pandas numpy scikit-learn
python analysis/generate_data.py     # writes data/*.csv
python analysis/analysis.py          # writes docs/dashboard_data.json + prints findings
python analysis/build_dashboard.py   # writes docs/index.html
# open docs/index.html in a browser
```

The T-SQL path: run `sql/schema.sql`, bulk-load the three CSVs, then `sql/analysis_queries.sql` — results cross-check against the Python.
